Introduction

During the development of web applications the backend and the frontend often rely on different teams.

So you need to create a common REST call definition agreement.

If the backend is developed first, utilities such as swagger or postman are probably sufficient for the subsequent development of the client.

When it is necessary to first define REST calls, virtual-service becomes a practical solution.

  • Server immediately available;
  • Only JavaScript on the backend;
  • Simplify procedures using lodash utilities;
  • Async functions to generate responses;
  • Untrusted code handled in dedicated processes with a predefined timeout;

Static values vs expressions

By default the value is taken as a literal string. To compute the value dynamically, prefix it with an equals sign = (Excel-style convention): everything after the = is evaluated as a JavaScript expression when the call is served.

  • application/json — literal value
  • =new Date().toISOString() — evaluated expression
  • =`Bearer ${headers.authorization}` — uses the request scope
functions

The fx icon next to the value input activates only when the value starts with =. Click it to open a full expression editor with syntax highlighting and contextual help.

Expressions share the same execution context as the call response: see Expression editor for the full list of available objects (db, pathValue, params, data, headers, cookies, _). If the expression fails the header/cookie is skipped.

In this box you can write JavaScript code to define the service's database object, which is available to all calls.

A timed function lets you repeat an operation at a set interval. A classic example: inserting notifications into the database every few seconds.

Function

Write JavaScript code to define the function that runs every interval seconds.

Expression editor

In the expression editor you write JavaScript code.

The return value can be a JSON object, a JavaScript object, or a Promise that resolves to one of those.

All code runs in a dedicated process with an execution timeout of 10 seconds.
If execution crashes or times out, an error is generated.

Available objects

The following objects are available in the JS context:

  • _: lodash utilities
  • db: the service database object (not in database definition)
  • samples: sample data (only in database definition)
  • pathValue: dynamic URL path parts (not in database definition)
  • params: query parameters from URL (not in database definition)
  • data: the request body (not in database definition)
  • headers: request headers (not in database definition)
  • cookies: request cookies (not in database definition)
Available samples

These objects are available only in database expression context:

  • samples.northwind: all Northwind database tables;
  • samples.italia: all Italian regions and municipalities;
  • samples.nations: all countries and nations;
